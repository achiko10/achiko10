# B2B საბითუმო პორტალის სერვერული არქიტექტურა

## 1. ტექნოლოგიური სტეკი
*   **ბექენდი (Django / Python):** განკუთვნილია მილიონობით ჩანაწერის (SKU), რთული საფასო პოლიტიკისა და პარტნიორული ხელშეკრულებების სამართავად.
*   **მონაცემთა ბაზა:** PostgreSQL (Master/Slave რეპლიკაციით დიდი დატვირთვისთვის).
*   **Live Inventory (Firebase Realtime DB):** როდესაც ერთი პარტნიორი ყიდულობს 1000 ერთეულ პროდუქტს, საწყობის მარაგის ნაშთი რეალურ დროში (წამიერად) იცვლება ყველა სხვა პარტნიორის ეკრანზე.
*   **ფრონტენდი:** Pure HTML/JS/CSS "Bioluminescent Scientific" დიზაინის ზუსტი ასახვისთვის.

## 2. ERP ინტეგრაცია (API)
*   ბექენდი მუდმივ რეჟიმში (Cron Jobs, Celery) უკავშირდება კომპანიის მთავარ ERP სისტემას (მაგ: SAP, Microsoft Dynamics 365 ან 1C).
*   **Sync Logic:** ყოველ 5 წუთში ხდება მარაგების სინქრონიზაცია (Pull), ხოლო ნებისმიერი გაფორმებული შეკვეთა მომენტალურად იგზავნება ERP-ში (Push) ინვოისირებისა და ლოჯისტიკისთვის.

## 3. კალათის რთული ლოგიკა (B2B Cart Engine)
სტანდარტული B2C (საცალო) კალათისგან განსხვავებით, B2B კალათა ითვალისწინებს:
*   **დინამიურ ფასდაკლებებს:** 
    *   Tier 1 პარტნიორი: -20%
    *   Tier 2 პარტნიორი: -10%
    *   მოცულობითი ფასდაკლება: თუ იღებს 500+ ერთეულს კონკრეტულ SKU-ზე, ფასი ავტომატურად მცირდება.
*   **საკრედიტო ლიმიტი (Credit Terms):** კომპანიებს შეუძლიათ შეიძინონ პროდუქცია ნისიად (Net 30, Net 60). კალათა გადახდისას ამოწმებს პარტნიორის დარჩენილ საკრედიტო ლიმიტს.
*   **მრავალსაწყობიანი ლოგიკა (Multi-warehouse):** კალათა უჩვენებს პარტნიორს, თუ რომელ რეგიონალურ საწყობშია პროდუქტი ხელმისაწვდომი.
