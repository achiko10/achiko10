# ექსპორტის და ელ-კომერციის არქიტექტურა (ღვინო / აგრო)

## 1. მულტილინგვური სტრუქტურა და SEO (Django CMS)
ვინაიდან კომპანია ორიენტირებულია ექსპორტზე, პლატფორმა მოითხოვს რთულ სტრუქტურას:
*   **ენა და ლოკალიზაცია:** `django-modeltranslation` ინტეგრაცია. თითოეულ პროდუქტს აქვს თარგმანის მოდელი (Title_en, Title_zh, Title_de). 
*   **SEO ოპტიმიზაცია:** ფრონტენდზე (HTML) გამოიყენება `hreflang` ტეგები, Canonical URLs და Open Graph მეტა-მონაცემები. URL სტრუქტურა იყოფა რეგიონების მიხედვით (მაგ: `/en-us/wine/`).

## 2. ტრანსსასაზღვრო მიწოდება და Order Processing
*   **ლოჯისტიკური API (DHL/FedEx):** კალათაში (Checkout) მომხმარებელი ირჩევს ქვეყანას და სისტემა რეალურ დროში ითვლის ტრანსპორტირების ღირებულებას მოცულობისა და წონის (Volumetric weight) მიხედვით.
*   **საბაჟო და საგადასახადო:** ბექენდი ინახავს HS Code-ებს თითოეული პროდუქტისთვის და ავტომატურად ითვლის სავარაუდო DDP (Delivered Duty Paid) ან DAP პირობებს B2B კლიენტებისთვის.

## 3. მონაცემთა ბაზა (Product DB)
*   **კატეგორიზაცია:** `Category` (ღვინო, თაფლი), `Variety` (საფერავი, რქაწითელი), `Vintage` (მოსავლის წელი).
*   **მარაგები (Inventory):** მრავალსაწყობიანი ლოგიკა. სისტემა ამოწმებს ლოკალურ მარაგს (საქართველო) და ექსპორტირებულ მარაგს პარტნიორ ჰაბებში (მაგ: EU Warehouse).
