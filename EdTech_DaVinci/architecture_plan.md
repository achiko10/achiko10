# EdTech პლატფორმის პრემიუმ არქიტექტურა

## 1. ტექნოლოგიური სტეკი
*   **ბექენდი (Django):** მძლავრი, სარელაციო მონაცემთა ბაზის მართვა. განკუთვნილია კურსების კატალოგიზაციისთვის, სტუდენტების აკადემიური მოსწრების აღრიცხვისა და ფინანსური ტრანზაქციების (Stripe / TBC / BOG) უსაფრთხოდ დასამუშავებლად.
*   **რეალურ დროში ინტეგრაცია (Firebase):** Push შეტყობინებები სტუდენტებისთვის ახალი დავალებების, ლაივ-სესიების დაწყებისა და ქულების დაწერის შესახებ. რეალური დროის ლაივ-ჩატი პედაგოგსა და სტუდენტს შორის.
*   **ფრონტენდი (Vanilla JS/HTML/CSS):** უმაღლესი წარმადობის ინტერფეისი "Da Vinci Cyber Blueprint" დიზაინით.

## 2. მომხმარებელთა პანელების სტრუქტურა

### A. სტუდენტის პანელი (Student Dashboard)
*   **Progress Tracker:** ინტეგრირებული გეომეტრიული დიაგრამები, რომლებიც აჩვენებს გავლილი მასალის პროცენტულ მაჩვენებელს.
*   **My Courses:** აქტიური კურსები, ვიდეო-ლექციებზე წვდომა და სასწავლო რესურსები (PDF, სქემები).
*   **Assignments & Quizzes:** დავალებების ატვირთვის მოდული და ონლაინ ტესტირება.

### B. მასწავლებლის ადმინკაბინეტი (Instructor Panel)
*   **Analytics:** სტუდენტების ჯგუფური და ინდივიდუალური აკადემიური მოსწრების რთული გრაფიკები (D3.js / Chart.js ინტეგრაციისთვის).
*   **Content Management:** ლექციების დამატების, ტესტების შექმნისა და დავალებების შეფასების სისტემა (Rich Text Editor).
*   **Revenue Dashboard:** გაყიდული კურსების სტატისტიკა და ტრანზაქციების ისტორია.

## 3. Decision-Table ლოგიკაზე აწყობილი სავარჯიშოების სისტემა
*   **კონცეფცია:** ნაცვლად სტანდარტული (A, B, C, D) ტესტებისა, სტუდენტს ეძლევა რთული პრობლემა და რამდენიმე ცვლადი.
*   **მოდელირება Django-ში:** 
    *   `Exercise`: სავარჯიშოს აღწერა.
    *   `DecisionVariable`: ცვლადები, რომლებსაც სტუდენტი მართავს.
    *   `OutcomeMatrix`: (Decision-table). ბექენდზე იქმნება მატრიცა, სადაც სხვადასხვა ცვლადის კომბინაცია იძლევა კონკრეტულ შედეგს (Outcome).
*   **ფრონტენდის ლოგიკა:** სტუდენტი აგებს "კიბერ-ნახაზს" ან ირჩევს გეომეტრიულ კავშირებს. JS აგზავნის JSON მონაცემებს ბექენდზე, სადაც ის დადარდება `OutcomeMatrix`-ს. 
*   **შეფასება:** სისტემა ავტომატურად აფასებს არა მხოლოდ საბოლოო პასუხს, არამედ "გადაწყვეტილების მიღების გზას" (Decision Path).
