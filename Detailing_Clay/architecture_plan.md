# Auto Detailing პრემიუმ პლატფორმის არქიტექტურა

## 1. ტექნოლოგიური სტეკი
*   **ბექენდი (Django):** მძლავრი ლოგიკა დინამიური ფასწარმოქმნისთვის.
*   **მონაცემთა ბაზა (PostgreSQL):** კლიენტების ისტორიის (CRM) და სერვისების აღრიცხვა.
*   **ფრონტენდი (Vanilla JS/HTML/CSS):** სუფთა, წარმადობაზე ორიენტირებული ანიმაციები "3D Clay" პრემიუმ ესთეტიკით.

## 2. დინამიური ფასწარმოქმნა (Dynamic Calculation)
Django ბექენდზე იქმნება ლოგიკა, სადაც ფასი იცვლება ავტომობილის კატეგორიის მიხედვით:
*   `VehicleCategory`: სედანი (x1.0 ფასი), კროსოვერი (x1.2), ჯიპი/SUV (x1.4).
*   `ServiceType`: კერამიკული დაფარვა, ქიმწმენდა, პოლირება.
*   **ალგორითმი:** მომხმარებელი ირჩევს ავტომობილის ზომას და ფრონტენდი (JS) რეალურ დროში ითვლის ფასს, ხოლო ჯავშნის გაფორმებისას Django ამოწმებს (Validate) ამ ფასს ბაზასთან.

## 3. მომხმარებლის ნაკადის მართვა (Workflow Module)
ბექენდზე ინტეგრირებულია სერვისის სტატუსების მოდული (Customer Flow):
1.  **Pending:** ჯავშანი შემოვიდა სისტემაში.
2.  **In Bay:** ავტომობილი შეყვანილია ბოქსში (სისტემა ავტომატურად უგზავნის SMS-ს კლიენტს).
3.  **Detailing:** მიმდინარეობს მუშაობა.
4.  **Ready for Pickup:** სამუშაო დასრულდა (Push შეტყობინება ინვოისთან ერთად იგზავნება კლიენტთან).
