# პრემიუმ ავტოიმპორტის პლატფორმის არქიტექტურა

## 1. ტექნოლოგიური სტეკი (Tech Stack)
*   **ბექენდი:** Django (Python) - არჩეულია მისი მძლავრი ORM-ის, უსაფრთხოების სტანდარტებისა და რთული ლოგიკების (ფასთწარმოქმნა, მომხმარებელთა როლები) მარტივად მართვის გამო.
*   **მონაცემთა ბაზა:** PostgreSQL - სარელაციო ბაზა ავტომობილების, ტრანზაქციებისა და მომხმარებლების მონაცემების უსაფრთხო შენახვისთვის.
*   **ფრონტენდი:** Pure HTML5, CSS3, Vanilla JavaScript - უზრუნველყოფს მაქსიმალურ სისწრაფეს, SEO ოპტიმიზაციას და დამოკიდებულებების (dependencies) მინიმალურ რაოდენობას. "Titanium Minimalism" დიზაინის ზუსტი ასახვისთვის.
*   **რეალურ დროში კომუნიკაცია:** Firebase Realtime Database / Cloud Messaging - ტრანსპორტირების სტატუსის განახლებებისა და აუქციონის ლაივ-შეტყობინებებისთვის.
*   **ქეშირება:** Redis - API გამოძახებების (Copart/IAA) და კალკულატორის შედეგების სწრაფი ქეშირებისთვის.

## 2. API ინტეგრაციის ლოგიკა (Copart / IAA)
*   **მონაცემთა სინქრონიზაცია (Cron Jobs / Celery):** 
    *   ყოველდღიური სინქრონიზაცია ლოტების მონაცემების გასაახლებლად (RESTful API / SOAP).
    *   `AutoLot` მოდელი Django-ში, რომელიც ინახავს უნიკალურ `lot_number`-ს, სურათების ლინკებს, დაზიანების ტიპს და მიმდინარე ფასს.
*   **Live Bidding (ცოცხალი ვაჭრობა):**
    *   Firebase-ის მეშვეობით მომხმარებლის კაბინეტში რეალურ დროში აისახება აუქციონის სტატუსი.

## 3. ფასთწარმოქმნის კალკულატორი (Pricing Engine)
*   **ალგორითმი:** ითვალისწინებს ავტომობილის ტიპს, ძრავის მოცულობას, გამოშვების წელს (განბაჟების გამოსათვლელად), აუქციონის მოსაკრებელს (Copart/IAA fee tables) და შიდა ტრანსპორტირების ფასებს შტატების მიხედვით.
*   **API ენდპოინტი:** `POST /api/v1/calculator/` იღებს VIN კოდს ან პარამეტრებს და აბრუნებს სრულ ხარჯთაღრიცხვას.

## 4. კონტეინერების თრექინგის სისტემის სტრუქტურა
*   **სტატუსების მართვა:** Django-ს ადმინ პანელიდან ან ლოგისტიკური პარტნიორის API-დან მიიღება კონტეინერის ნომერი (Container ID).
*   **მარშრუტი:** 
    1. აუქციონიდან პორტამდე (Land Transit).
    2. პორტში ჩატვირთვა (Loaded on Vessel).
    3. ოკეანეში ტრანზიტი (Ocean Transit - ინტეგრაცია საზღვაო ხაზების API-სთან, მაგ: Maersk, MSC).
    4. დანიშნულების პორტში ჩამოსვლა (Arrived).
*   **შეტყობინებები:** Firebase აგზავნის Push შეტყობინებას მომხმარებლის კაბინეტში სტატუსის ცვლილებისას.

## 5. მომხმარებლის პერსონალური კაბინეტის არქიტექტურა
*   **ავტორიზაცია:** JWT (JSON Web Tokens) Django Rest Framework-ის მეშვეობით.
*   **Dashboard მოდულები:**
    *   *My Vehicles:* შეძენილი ან საყიდლად გამზადებული მანქანების სია.
    *   *Live Tracking:* ინტერაქტიული რუკა (Mapbox/Google Maps) და თაიმლაინი კონტეინერის ლოკაციით.
    *   *Finance & Invoices:* ინვოისების გენერაცია (PDF) და გადახდების ისტორია.
    *   *Saved Lots:* ფავორიტებში დამატებული ლოტების მონიტორინგი.
